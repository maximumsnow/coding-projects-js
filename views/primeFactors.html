If you completed this challenge in C# you would have worked with a class. In JavaScript classes are uncommon so they have not been used in this solution.

In ./lib/primeFactors.js there are 2 funcitons. In the getPrimeFactors function code needs to be added that will find all of the prime factors for the integer and return a string that represents all of the factors with a "x" beween the primes so it looks like a math problem, with an equals "=" symbol and then the number that was passed in.

For example...

Given: A user wants to know the prime factors for 6
When: They use the URL http://localhost:3000/primeFactors?number=6
Then: 2 x 3 = 6 is returned

Given: A user wants to know the prime factors for 13
When: They use the URL http://localhost:3000/primeFactors?number=13
Then: 13 = 13 is returned

Given: A user wants to know the prime factors for 16
When: They use the URL http://localhost:3000/primeFactors?number=6
Then: 2 x 2 x 2 x 2 = 16 is returned

The getPrimeFactors function takes the number you are passing in and assembles an array. This is a list of integers. Think of a train. These train cars can hold one number. If you want to add another car to the train to hold a number, you add one more. Together they become a train of increasing length each time you add another number. That’s what we are doing here. The code here creates the variable of type array called primeFactors. We can then add numbers to this array. The core of this challenge will be here, in this function. We have to write code that will take any number and return it’s prime factors.

Unit Tests have been added for numbers 1-17. Once it is working you should try adding 18-21 and a few large numbers.

We’ve taken up the complexity with this challenge so it’s going to take some time. If you get stuck, Google Prime Factors Kata. You’ll find some videos that can help. Or ask your peers, or myself of course.
